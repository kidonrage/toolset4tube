# YouTube Curator v0

## Goal

Build a local, free, boring tool that helps an AI agent triage a YouTube playlist.

The tool does not delete, move, or mutate anything on YouTube. It only collects evidence,
caches it locally, and writes reports that an agent can use.

## Non-Goals

- No YouTube Data API.
- No OAuth.
- No backend server.
- No MCP server.
- No web UI.
- No queue, Redis, Postgres, object storage, or workers.
- No automatic deletion.
- No mass transcription.

## Hard Rules

- Use `yt-dlp` as the only YouTube fetcher.
- Store state locally in SQLite.
- Store larger text artifacts as files.
- Never download video.
- Fetch subtitles only for videos selected for deeper processing.
- Never transcribe audio unless the user explicitly asks later.
- Every run must be resumable.
- Every network failure must leave cached data intact.
- Reports must separate metadata-based judgment from transcript-based judgment.

## Data Layout

```text
data/
  state.sqlite
  metadata/
    <video_id>.json
  subtitles/
    <video_id>.<lang>.vtt
  reports/
    report.md
    decisions.jsonl
    delete-candidates.csv
```

## Commands

### Scan Playlist

```bash
youtube-curator scan PLAYLIST_URL --limit 50 --offset 0
```

Fetches playlist entries and basic metadata through `yt-dlp`.

Output:

- updates `data/state.sqlite`
- writes raw metadata snapshots into `data/metadata/`

### Build Agent Context

```bash
youtube-curator context --profile profile.example.yaml --limit 50
```

Writes a compact Markdown/JSON context file for the AI agent.

The context should include:

- title
- URL
- channel
- duration
- publish date if available
- description excerpt
- chapters if available
- subtitle availability if already known
- previous decision if any

### Fetch Subtitles

```bash
youtube-curator subtitles --video-id VIDEO_ID --langs ru,en
```

Fetches manual and auto subtitles for one selected video.

This is not part of the default scan. Subtitles are expensive in time and noise, even when
they are free in money.

### Write Report

```bash
youtube-curator report
```

Combines stored decisions and artifacts into a human-readable report.

## Decision Types

```text
WATCH_FULLY
WATCH_PARTIAL
SUMMARY_ENOUGH
DELETE
LOW_CONFIDENCE
```

## Decision Record

Each AI decision is stored as one JSON object per line:

```json
{
  "video_id": "abc123",
  "decision": "WATCH_PARTIAL",
  "confidence": 0.78,
  "basis": "metadata",
  "reason": "Relevant to AI agents, but only one section matches the current profile.",
  "watch_ranges": [
    {
      "start": "12:30",
      "end": "28:10",
      "reason": "Agent workflow architecture"
    }
  ],
  "summary": null,
  "created_at": "2026-06-21T00:00:00Z"
}
```

## v0 Flow

```text
1. User provides playlist URL and priority profile.
2. CLI scans N playlist items through yt-dlp.
3. CLI writes local metadata cache.
4. CLI builds compact agent context.
5. Agent classifies videos using the profile.
6. Agent writes decisions.jsonl.
7. CLI renders report.md and CSV helper files.
8. User manually opens YouTube and removes videos if desired.
```

## Failure Model

If `yt-dlp` fails:

- keep old cached metadata;
- mark the current item as failed;
- continue with the next item when possible;
- write the failure reason into SQLite;
- never corrupt existing report files.

If the AI agent fails:

- existing scan data remains valid;
- rerun only the decision step;
- do not rescan YouTube unless explicitly requested.

## First Implementation Slice

The first useful version needs only:

- `scan`
- `context`
- `report`
- SQLite schema
- profile YAML
- JSONL decisions

Subtitles can come after the metadata-only loop is proven useful.
